vt(4) enabled by default

October 19, 2014 Jean-Sébastien Pédron

1 minute read

vt(4), the new console driver, is enabled out-of-the-box in FreeBSD 11-CURRENT (the development branch of FreeBSD) as of today!

For those who don’t know it yet, vt(4) brings important features:

  • It supports Unicode and double-width characters. It finally allows to use an UTF-8 locale.
  • It integrates with the new kernel video drivers. That means that you can switch from an X session to the console and back. syscons(4) was never taught about the KMS drivers, and this was a major regression for users because the screen was blank when vt-switching or terminating the X server.
